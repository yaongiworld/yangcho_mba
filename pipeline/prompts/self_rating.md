# Self-rating prompt — drives the confidence gate

PLACEHOLDER. Implementation lands in W3 alongside the friction analyzer.

## What this prompt does

Given a freshly-generated friction analysis, rate it 1–10 on:
  - mechanism plausibility
  - alignment with the hero case studies' reasoning style
  - specificity of the friction observation

A single combined score. ≥8 → auto-publish. <8 → queue for Yangcho's review.

The self-rating is generated by the SAME model that generated the analysis,
in a separate call. This is deliberately weak as a quality signal (a confused
model can be confidently wrong) but it's the cheapest filter, and Yangcho's
retract-anytime override is the safety net.

## Output schema

JSON:
```json
{
  "rating": "integer 1–10",
  "reasoning": "string — one paragraph on what would make this stronger"
}
```

# Variables

- `{{friction_summary}}`
- `{{friction_mechanism}}`
- `{{efficacy_class}}`
- `{{hero_case_excerpt}}` — short excerpt of one hero case for stylistic comparison
